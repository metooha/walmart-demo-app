# TextArea Component

## Overview

The TextArea component is a multi-line text input field built with Living Design 3.5 semantic tokens. It supports various states, sizes, and features including labels, error messages, helper text, character counting, and AI-generated content indicators.

## When to Use

- When users need to enter or edit multiple lines of text
- For collecting detailed feedback, descriptions, or comments
- When you need to show character count limits
- For displaying AI-generated content that users can edit

## Component API

### Props

#### Required Props

| Prop | Type | Description |
|------|------|-------------|
| `label` | `React.ReactNode` | The label for the textarea (required for accessibility) |

#### Optional Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | `''` | The current value of the textarea |
| `onChange` | `(event) => void` | - | Callback when the value changes |
| `size` | `'large' \| 'small'` | `'large'` | The size of the textarea (large: 128px, small: 100px) |
| `disabled` | `boolean` | `false` | If the textarea is disabled |
| `readOnly` | `boolean` | `false` | If the textarea is read-only |
| `isMagic` | `boolean` | `false` | Shows magic icon for AI-generated content |
| `helperText` | `React.ReactNode` | - | Helper text to display below the textarea |
| `error` | `React.ReactNode` | - | Error message (replaces helper text when present) |
| `maxLength` | `number` | - | Maximum character length (shows counter) |
| `id` | `string` | auto-generated | ID for the textarea element |
| `a11yMagicLabel` | `string` | `'AI Generated'` | Accessible label for the magic icon |
| `maxLengthA11yAnnouncement` | `string` | - | Custom announcement for character count |
| `textAreaProps` | `object` | - | Additional textarea HTML attributes |
| `UNSAFE_className` | `string` | - | Override className (use sparingly) |
| `UNSAFE_style` | `object` | - | Override style (use sparingly) |

## Usage Examples

### Basic TextArea

```tsx
import { TextArea } from '@/components/ui/TextArea';

function MyForm() {
  const [value, setValue] = React.useState('');

  return (
    <TextArea
      label="Description"
      value={value}
      onChange={(e) => setValue(e.target.value)}
      textAreaProps={{ placeholder: 'Enter description...' }}
    />
  );
}
```

### With Helper Text

```tsx
<TextArea
  label="Comments"
  value={value}
  onChange={(e) => setValue(e.target.value)}
  helperText="Provide detailed feedback about the product"
/>
```

### With Error State

```tsx
<TextArea
  label="User Input"
  value={value}
  onChange={(e) => setValue(e.target.value)}
  error="This field is required"
/>
```

### With Character Counter

```tsx
<TextArea
  label="Bio"
  value={value}
  onChange={(e) => setValue(e.target.value)}
  maxLength={200}
  helperText="Tell us about yourself"
/>
```

### AI-Generated Content

```tsx
<TextArea
  label="AI Response"
  value={aiGeneratedText}
  onChange={(e) => setAiGeneratedText(e.target.value)}
  isMagic
  helperText="This content was generated by AI"
/>
```

### Small Size Variant

```tsx
<TextArea
  label="Quick Note"
  value={value}
  onChange={(e) => setValue(e.target.value)}
  size="small"
  helperText="For condensed layouts"
/>
```

### Disabled State

```tsx
<TextArea
  label="Read Only Info"
  value="This cannot be edited"
  disabled
/>
```

### Read-Only State

```tsx
<TextArea
  label="Terms"
  value={termsText}
  readOnly
/>
```

## Accessibility

### ARIA Attributes

The TextArea component implements the following accessibility features:

1. **Label Association**: Labels are properly associated with textareas using `htmlFor` and `id`
2. **Error Messages**: Connected via `aria-describedby` for screen reader announcements
3. **Character Count**: Live region with debounced announcements (1.5s delay)
4. **Magic Icon**: Includes `aria-label` for AI-generated content indicator
5. **Invalid State**: `aria-invalid="true"` when error is present

### Keyboard Navigation

- **Tab**: Focus the textarea
- **Shift + Tab**: Focus previous element
- **Standard text editing**: All native textarea keyboard shortcuts work

### Screen Reader Behavior

- Label is announced when textarea receives focus
- Error messages are announced immediately
- Character count updates are announced with 1.5s debounce to avoid spam
- Magic icon is announced as "AI Generated" (customizable)

## States

### Enabled (Default)

Normal interactive state with standard border and background.

### Hovered

Activated when cursor is over the textarea. Shows thicker border (2px) and different background.

### Focused

Activated when textarea has keyboard focus. Shows thicker border (2px) and different background.

### Disabled

Non-interactive state with reduced opacity. Cursor changes to `not-allowed`.

### Read-Only

Non-editable state but users can select and copy text. Visual style similar to enabled state.

### Error

Indicates validation errors with red border and background. Error text appears below with icon.

### Magic (AI-Generated)

Indicates AI-generated content with special border color and magic icon next to label.

## Size Variants

### Large (Default)

- **Height**: 128px
- **Padding**: 16px
- **Font**: `ld-semantic-font-body-medium-*`
- **Use**: Standard forms and layouts

### Small

- **Height**: 100px
- **Padding**: 8px (vertical) 12px (horizontal)
- **Font**: `ld-semantic-font-body-small-*`
- **Use**: Condensed layouts, typically on desktop

## Best Practices

### DO

✅ Always provide a descriptive label
✅ Use helper text to provide context or instructions
✅ Show character counter for fields with length limits
✅ Use error messages to guide users to fix validation issues
✅ Set `isMagic` to true for AI-generated content
✅ Use read-only state for content that shouldn't be edited but can be selected

### DON'T

❌ Don't use placeholder text as a replacement for labels
❌ Don't make error messages vague or unhelpful
❌ Don't disable textareas when you mean read-only
❌ Don't override LD semantic tokens with hard-coded values
❌ Don't use textareas for single-line input (use Input component instead)

## Migration from Old Component

If you're migrating from the old Shadcn textarea component:

### Old (Shadcn)

```tsx
import { Textarea } from '@/components/ui/textarea';
import { Label } from '@/components/ui/label';

<div>
  <Label htmlFor="message">Your message</Label>
  <Textarea id="message" placeholder="Type here..." />
</div>
```

### New (LD 3.5)

```tsx
import { TextArea } from '@/components/ui/TextArea';

<TextArea
  label="Your message"
  value={value}
  onChange={(e) => setValue(e.target.value)}
  textAreaProps={{ placeholder: 'Type here...' }}
/>
```

### Key Changes

1. **Integrated Label**: Label is now part of the component API
2. **Controlled Component**: Use `value` and `onChange` props
3. **Enhanced Features**: Built-in error, helper text, and counter support
4. **LD 3.5 Tokens**: All styling uses semantic tokens
